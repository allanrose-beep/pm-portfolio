Daily Pet-Parent Nudges: bite-sized education through cat facts

TypeFeature Concept
ToolsN8N · Cat Facts API
GoalMicro-engagement through light education

The problem we couldn't ignore

While pet ownership is rising globally, many pet parents engage more emotionally than educationally with their animals. In research across pet communities and forums, a clear pattern emerged: owners adore their pets but rarely seek new information about them.

This creates an awareness gap. A lot of care mistakes happen not from negligence, but from simply not knowing better. Yet asking users to read articles or complete onboarding lessons felt heavy-handed. The question became: can we lightly educate pet owners through something delightful and unobtrusive?

Solution: fun, bite-sized education with cat facts

The proposed feature introduces cat facts into a pet app — delivered either as push notifications or embedded on the login screen. These facts are quirky, scientific, or behavioural tidbits designed to:

  • Delight users and increase app stickiness
  • Nudge pet owners to learn something new each day
  • Reinforce daily positive engagement with the app

Prototype: workflow automation using N8N

To quickly test this idea, I built a working prototype using N8N, an open-source workflow automation tool. The workflow triggers at a set daily interval, connects to a public Cat Facts API, and receives a random fact as a JSON response. That data is then processed, cleaned, and formatted into a readable message pushed directly to a Gmail inbox — a fresh cat fact, every day.

This was a practical way to test feasibility and play with content cadence before committing to any in-app development.

Scaling to a real product feature

While the prototype delivers to a single inbox, the core logic is ready to scale. Here's how it would sit inside a real app:

  • Trigger a daily push notification with a random cat fact
  • Tie delivery timing to user-set preferences
  • Surface facts on the dashboard or post-login screen
  • Show facts on loading screens for passive, no-friction discovery

What's next

The goal is to evolve this prototype into a real app feature and measure its effect on retention and daily active usage — creating micro-engagement hooks that are low-effort, high-delight, with the added bonus of passive education.

Even small features can make a big difference when they come from genuine user insight and are tested with real workflows.
